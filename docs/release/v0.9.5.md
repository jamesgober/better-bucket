# better-bucket v0.9.5 — Release Candidate

**The last patch before 1.0.** v0.9.5 is documentation polish only: the design
doc that was missing, a README brought in line with CI, and nothing else. No
code changed, no API moved, and the `0.9.1`–`0.9.4` bug-fix slots stayed empty
because nothing needed fixing. After a short stability soak, `1.0.0` follows.

## What is better-bucket?

A genuinely better token bucket for Rust. The hot path — `try_acquire` — is
lock-free (one `compare_exchange_weak` on a packed atomic word),
allocation-free, and cache-line aligned, with a division-free fixed-point refill
computed lazily from an injected monotonic clock. It never over-grants, the API
is frozen, and token bucket is its sole job — it is the primitive `rate-net`
consumes.

## What's new in 0.9.5

### `docs/DESIGN.md`

The architecture and design-decision reference that the surface docs (`API.md`)
and the numbers (`BENCHMARKS.md`) did not cover:

- the packed `AtomicU64` layout — tokens in millitokens, milliseconds in the low
  half — and why a single word matters (it is what makes the acquire path one
  CAS, and what keeps two-atomic lost-update races out);
- the CAS acquire loop and why its memory ordering is `Relaxed`;
- the division-free `Q22` fixed-point refill, derived from the period in
  nanoseconds so sub-millisecond rates stay accurate;
- the wrapping millisecond clock and why it replaced a saturating one (no
  long-uptime stall);
- overflow safety, and how the no-over-grant contract is defended by `loom`, the
  stress test, the allocation audit, `proptest`, and the adversarial suite;
- why the monotonic clock is the performance floor, not the bucket;
- and the recorded decisions: token-bucket-only, plain `u32` over `NonZeroU32`,
  no keyed store, zero `unsafe`, no SIMD.

### README

Testing commands now match CI — default features rather than `--all-features`
(which would pull the benchmark-only `governor`), plus the `no_std`, doc, and
`cargo deny` / `cargo audit` gates. A new Documentation section links the
reference, design, benchmark, and changelog pages.

## Verification

Run on Windows x86_64 and on Linux x86_64 (WSL2 Ubuntu), Rust stable 1.95.x and
MSRV 1.85; the MSRV build is also enforced by the CI matrix. All green:

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --no-default-features -- -D warnings
cargo test
cargo test --no-default-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps
RUSTFLAGS="--cfg loom" cargo test --test loom_acquire
cargo deny check
cargo audit
```

Counts at this tag (default features): 34 unit tests + 1 allocation audit + 13
hardening tests + 4 consumer-pattern tests + 4 property tests + 27 doctests;
`loom`: 1 model check. `cargo deny check` and `cargo audit` are clean.
