# better-bucket v0.9.0 — Beta

**Stabilization, not change.** v0.9.0 opens the beta band: broaden the test net,
re-confirm the numbers, and otherwise leave the frozen surface alone. No bug
fixes were needed and no API moved.

## What is better-bucket?

A genuinely better token bucket for Rust: lock-free, allocation-free, never
over-grants, API frozen. Token bucket is its sole job; it is the primitive
`rate-net` consumes.

## What's new in 0.9.0

### A stronger property: the retry hint is honest

The acquire path returns a `retry_after` on denial. A new property test,
`retry_after_is_an_honest_lower_bound`, checks that the hint never
under-promises: across a wide space of rates and request sizes, draining a
bucket, reading the `retry_after` from the denial, advancing the clock by
exactly that amount, and retrying always succeeds.

This matters because two pieces of fixed-point arithmetic have to agree. The
denial's wait is computed with a **ceiling** (`time_for`), while refill accrues
with a **floor** (the `Q22` multiply-and-shift). If those disagreed by even one
unit, a caller that waited exactly as long as told could still be denied. The
property proves they agree.

### Benchmarks re-confirmed

A re-run after the `0.7` wrap change held or improved every repeatable metric —
single-thread `try_acquire`, the algorithm-only figure, and refill-after-idle —
with no regression against the `0.6` baseline. The contended figures are noted
in `docs/BENCHMARKS.md` as host-sensitive (the bench takes the slowest thread's
time, which a non-isolated machine perturbs run-to-run); they show lock-free
scaling rather than a precise constant.

## Breaking changes

**None.** A test and a docs note; the public surface is unchanged and frozen.

## Verification

Run on Windows x86_64 and on Linux x86_64 (WSL2 Ubuntu), Rust stable 1.95.x and
MSRV 1.85; the MSRV build is also enforced by the CI matrix.

```bash
cargo fmt --all -- --check
cargo clippy --all-targets -- -D warnings
cargo clippy --all-targets --no-default-features -- -D warnings
cargo test
cargo test --no-default-features
RUSTDOCFLAGS="-D warnings" cargo doc --no-deps
RUSTFLAGS="--cfg loom" cargo test --test loom_acquire
cargo deny check
cargo audit
```

Counts at this tag (default features): 34 unit tests + 1 allocation audit + 13
hardening tests + 4 consumer-pattern tests + 4 property tests + 27 doctests;
`loom`: 1 model check. `cargo deny check` and `cargo audit` are clean.
